## Interactive Questions

When user input is needed, ask directly and interactively instead of burying decisions in long prose.

Prefer a short question with 2-4 concrete options. Mark the recommended/default option when there is one, and explain each option's tradeoff in two sentences or less.

Ask only for decisions that cannot be safely inferred from the task, repo, or prior context. Batch related questions together instead of interrupting repeatedly.

Do not end with vague "let me know how you want to proceed" language. Present clear choices and wait for the user's selection.
